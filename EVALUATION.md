## 2. معمارية التوجيه (Routing Architecture)

### 2.1 استراتيجية Qwen-First
```
كل استدعاء AI → محاولة Qwen (أساسي) → Qwen احتياطي → Gemini احتياطي أخير
                                                           ↑
                                          TTS فقط = Gemini مباشرة
```

### 2.2 خريطة المهام ← النماذج

| نوع المهمة | النموذج الأساسي | النموذج الاحتياطي | الوكلاء |
|------------|-----------------|-------------------|---------|
| `text_fast` | qwen3.5-flash | qwen-flash | مراقبة، تنبيهات |
| `text_complex` | qwen3-max-2026-01-23 | qwen3.5-397b | بيليه، المخرج، المنسق، NEXUS |
| `text_creative` | qwen3-max-2026-01-23 | qwq-plus | معمار القصة، السرد العاطفي |
| `code` | qwen3-coder-480b | qwen3-coder-plus | مخرج البرومبت، المنسق |
| `code_fast` | qwen3-coder-flash | qwen-flash | — |

## 3. نتائج الاختبار الشامل

### 3.2 اختبار معمار القصة (Story Architect)
```
المدخل:  "اكتب بداية مشهد لفيلم عربي عن رحلة مكتشف"
النموذج: gemini-2.5-flash-lite (fallback)
الرموز:  1220
النتيجة: "صدى الرمال" — مشهد متكامل بجودة عالية

```

### 3.3 اختبار endpoint النماذج
```
GET /api/settings/ai-models
النماذج الكلي: 27 نموذج
الأساسية: 8 نماذج
DashScope Seed: {"success":true,"message":"تم تحديث endpoint إلى DashScope القياسي"}
الحالة: ✅ ناجح
```

### 3.4 سبب استخدام Gemini كـ Fallback
الملاحظة: المفتاح المخزَّن في قاعدة البيانات (`api_key.alibaba`) هو مفتاح بيئة MaaS
مخصص للـ endpoint `ws-twcxat39x22mi7rg.ap-southeast-1.maas.aliyuncs.com` ولا يعمل
مع DashScope القياسي. النظام يعمل بشكل صحيح عبر Gemini كاحتياط.

**الحل للمستخدم:** إضافة مفتاح DashScope من [console.aliyun.com](https://console.aliyun.com) في
إعدادات → مفاتيح API → تحديث مفتاح Alibaba.

## 6. خطة التحسينات — Feature Enhancement Plan

### الأولوية 1 — حرجة (Sprint 1)
**[F-001] إضافة مفتاح DashScope الصحيح**
- الهدف: تفعيل Qwen كنموذج أساسي بدلاً من Gemini كاحتياط
- الخطوات: الحصول على `sk-xxxx` من DashScope console → تحديثه في الإعدادات
- التأثير: توفير 60-70% من تكلفة Gemini

**[F-002] نظام تتبع الاستهلاك الفعلي**
- تسجيل الاستهلاك الحقيقي من API responses (usage.total_tokens)
- تحديث جدول `model_quota` بعد كل استدعاء ناجح
- عرض تاريخ الاستهلاك اليومي/الأسبوعي

**[F-003] تنبيهات الرصيد المنخفض**
- إشعار عند وصول نموذج لـ 20% من رصيده
- WebSocket push للـ frontend
- البديل التلقائي يُفعَّل قبل النفاد الكامل

### الأولوية 2 — مهمة (Sprint 2)
**[F-004] لوحة مراقبة الوكلاء المباشرة**
- مخطط شبكي يُظهر الوكلاء النشطين في الوقت الحقيقي
- سرعة الاستجابة لكل وكيل
- عدد المهام المنجزة هذا الجلسة

**[F-005] تاريخ المحادثات متعدد الجلسات**
- حفظ محادثات بيليه في PostgreSQL
- استعادة السياق بين الجلسات
- بحث في سجل المحادثات

**[F-006] تصدير السيناريو بصيغة PDF/Fountain**
- تحويل ناتج معمار القصة إلى تنسيق Fountain (صيغة السيناريو القياسية)
- تصدير PDF مُنسَّق جاهز للإنتاج

### الأولوية 3 — تحسينات (Sprint 3)
**[F-007] معالجة الصور في اللوحة المصورة**
- رفع صورة → وكيل الرؤية يحللها ويقترح مشاهد
- دمج Qwen3-VL-235B لتحليل البصري

**[F-008] Streaming للردود الطويلة**
- Server-Sent Events لردود بيليه ومعمار القصة
- عرض الرد حرفاً بحرف بدلاً من انتظار الاكتمال

**[F-009] نظام قوالب المشاريع**
- قوالب جاهزة لأنواع الأفلام (وثائقي، درامي، تشويقي)
- حفظ واستعادة إعدادات وكيل معمار القصة

**[F-010] تحليل مقارن بين النماذج**
- إرسال نفس المهمة لـ Qwen و Gemini في آنٍ واحد
- عرض المقارنة جنباً إلى جنب
- تقييم الجودة بمقياس موحد
